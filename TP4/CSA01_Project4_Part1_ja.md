## a) 問題の説明

本プロジェクトでは、**Winner-Take-All (WTA) ネットワーク**（Kohonen ネットワークとも呼ばれる）を用いた教師なし競合学習を扱う。

目的は、学習パターンの分布をうまく表現できる少数の代表ベクトル（プロトタイプ）を見つけることである。教師あり学習とは異なり、クラスラベルを一切使用せず、ネットワークはデータの分布だけを手がかりに自己組織化する。

実験は以下の 2 つで構成される。

**基礎実験** (`kohonen_network.c`)  
単位円上に配置された 5 つの 2 次元単位ベクトルを M = 2 個のニューロンで分類する。入力が単位ノルムであり勝者選択に内積を用いるため、これは単位円上でデータの方向を最もよく代表する 2 点を求める問題と等価である。

**Iris データセット実験** (`kohonen_network_iris.c`)  
Fisher (1936) による Iris データセットは、3 種のアヤメの花の 150 サンプルからなる。各サンプルはがく片の長さ・幅、花弁の長さ・幅の 4 特徴量を持つ。M = 3 のニューロンを持つ WTA ネットワークを学習し、正解ラベルなしに 3 品種を教師なしで分離できるかを検証する。

## b) 手法と実装

### WTA 学習アルゴリズム

ネットワークは M 個の出力ニューロンで構成され、各ニューロン m は重みベクトル $\boldsymbol{w}_m \in \mathbb{R}^I$ を持つ。各学習ステップでは、入力 $\boldsymbol{x}^{(p)}$ に対して内積が最大のニューロン $m^*$ が勝者となる。

$$m^* = \underset{m}{\arg\max} \ \boldsymbol{w}_m^\top \boldsymbol{x}^{(p)}$$

更新されるのは勝者ニューロンの重みのみである。

$$\boldsymbol{w}_{m^*} \leftarrow \boldsymbol{w}_{m^*} + \alpha \left(\boldsymbol{x}^{(p)} - \boldsymbol{w}_{m^*}\right), \quad \alpha = 0.5$$

更新後は重みを再正規化して単位長を維持する。

$$\boldsymbol{w}_{m^*} \leftarrow \frac{\boldsymbol{w}_{m^*}}{\|\boldsymbol{w}_{m^*}\|}$$

重みと入力がともに単位ノルムのとき、内積はコサイン類似度に等しくなる。したがってこのルールは「入力の方向に最も近い方向を向いているニューロン」を勝者とすることを意味する。

### 入力の前処理

Iris 実験では、学習前にすべての入力ベクトルを L2 正規化する。

$$\boldsymbol{x}^{(p)} \leftarrow \frac{\boldsymbol{x}^{(p)}}{\|\boldsymbol{x}^{(p)}\|}$$

WTA の勝者選択は内積に基づくため、正規化なしでは特徴量の絶対値が大きいサンプルが常に有利になり、方向の近さではなくノルムの大きさで勝者が決まってしまう。L2 正規化はこのアルゴリズムを正しく機能させるために本質的な役割を持つ。

### 実装

基礎プログラムでは 5 つの単位ベクトルをコード内に定義する。

```c
double x[P][I] = {
    {0.8,    0.6   },
    {0.1736, -0.9848},
    {0.707,  0.707 },
    {0.342,  -0.9397},
    {0.6,    0.8   }};
```

Iris プログラムでは `iris.data` から 4 特徴量を読み込み、種名ラベルを除いて L2 正規化する。

```c
for (int p = 0; p < P; p++) {
    double n = 0;
    for (int j = 0; j < I; j++) n += x[p][j] * x[p][j];
    n = sqrt(n);
    for (int j = 0; j < I; j++) x[p][j] /= n;
}
```

### 発見したバグと修正

元の `kohonen_network_iris.c` には 2 つの問題があり、いずれも「全 150 パターンがクラス 0 に分類される」という症状を引き起こしていた。

**バグ 1 — 勝者スコアの初期値 `s0 = 0`**

```c
/* 元のコード（誤り） */
s0 = 0;

/* 修正後 */
s0 = -1e9;
```

`s0 = 0` の場合、あるニューロンが勝者になるには内積が厳密に正である必要がある。Iris の全特徴量は正値であり、L2 正規化後の入力ベクトルはすべて **4 次元の正のオクタント** に存在する。ランダム初期化で負の成分を持つニューロンは全入力に対して内積が負になり、一度も勝者になれない。こうしてニューロン 0 が序盤から独占し、他のニューロンは永久に排除される（**デッドニューロン問題**）。`s0 = -1e9` とすることで内積の符号によらず真の最大値が選ばれるようになる。

**バグ 2 — データの存在領域外への重み初期化**

```c
/* 元のコード（誤り） */
w[m][i] = (double)(rand() % 10001) / 10000.0 - 0.5;  /* [-0.5, 0.5] */
/* L2 正規化後は単位超球面全体に散らばる */

/* 修正後 */
int rp = (rand() % P);
for (i = 0; i < I; i++) w[m][i] = x[rp][i];
```

$[-0.5, 0.5]^4$ のランダム初期化（正規化後）では、重みベクトルが単位超球面全体に散らばる。Iris データは正のオクタント内にしか存在しないため、負の成分を持つニューロンは最初から体系的に不利な状況に置かれる。学習データからランダムサンプリングして初期化することで、全ニューロンがデータの実際の分布の中から出発でき、特定のニューロンだけが最初から有利という状況を避けられる。

## d) 追加問題の提起：勝者選択の基準はクラスタリング品質に影響するか？

### 背景と動機

基礎 WTA プログラムはコサイン類似度（L2 正規化後の内積）で勝者を選ぶ。この基準はすべてのサンプルを単位超球面上の方向ベクトルとして扱い、絶対的な大きさを無視する。

代替として、**ユークリッド距離**を勝者基準とする方法がある。これは標準的な k-means クラスタリングに相当し、特徴量の絶対値を考慮する。

我々が提起した問いは次のとおりである。**「2 つの基準のどちらが Iris の 3 品種をより正確に分離できるか？」**

2 つのアプローチはデータについて異なる前提を置いている。

- **コサイン類似度 WTA** — 特徴量の*比率*（例：がく片長に対する花弁長の割合）でクラスタリングする。L2 正規化が必須。
- **ユークリッド距離 WTA** — 特徴量の*絶対値*でクラスタリングする。正規化不要。

Iris の 3 品種は比率でも絶対値でも違いがあるため、どちらが向いているかは実際に試してみないとわからない。

## e) 手法の改良：ユークリッド距離ベースの WTA

`Practice/kohonen_euclidean.c` として、ユークリッド距離を勝者基準とする WTA の変種を実装した。

$$m^* = \underset{m}{\arg\min} \|\boldsymbol{x}^{(p)} - \boldsymbol{w}_m\|^2$$

入力の L2 正規化は行わない。重みは元の特徴量空間（cm 単位）のプロトタイプとして機能するため、更新後の再正規化も不要である。

$$\boldsymbol{w}_{m^*} \leftarrow \boldsymbol{w}_{m^*} + \alpha \left(\boldsymbol{x}^{(p)} - \boldsymbol{w}_{m^*}\right)$$

ユークリッド WTA は遠くのクラスターに向かってゆっくり移動する性質があるため、公平な比較のために学習率を $\alpha = 0.1$、エポック数を 50 に設定した。

勝者探索のコア部分：

```c
dist0 = 1e18;
for (m = 0; m < M; m++) {
    dist = 0;
    for (i = 0; i < I; i++)
        dist += (w[m][i] - x[p][i]) * (w[m][i] - x[p][i]);
    if (dist < dist0) { dist0 = dist; m0 = m; }
}
```

初期化方法（学習データからのランダムサンプリング）はコサイン WTA と同じにした。この初期化は勝者基準によらず有効なので、両手法で統一している。

### 考察

どちらの手法もセトサを完全に分類できた。セトサは他の 2 種と特徴空間上で明確に分離されており、どの基準でも容易に区別できる。

コサイン WTA はバージニカを 50/50 で完全分類するが、バーシカラーを 10 サンプル誤分類する。ユークリッド WTA はその逆で、バーシカラーを 47/50 で正しく分類できる一方、バージニカを 15 サンプル誤分類する。図 4 では、ユークリッド WTA の境界が右方向へシフトし、バージニカの一部がバーシカラークラスタに吸収されている様子が視覚的に確認できる。

この逆転現象の原因は、各基準がバーシカラーとバージニカの「距離」をどのように知覚するかの違いにある。

生の特徴量空間では、バージニカはすべての特徴量においてバーシカラーより大きな絶対値を持つ（花弁が長く、がく片も大きい）。バージニカのサンプルとバーシカラーの重心とのユークリッド距離が、全体的なサイズ差が小さい場合には縮まり、バージニカの一部がバーシカラーのクラスターに引き寄せられる。結果として、ユークリッド WTA のバージニカプロトタイプは最も花が大きいサンプル（35/50）のみを代表するにとどまる。

一方 L2 正規化後は、絶対的な大きさではなく特徴量の*比率*が類似度の判断基準になる。バージニカはバーシカラーに比べてがく片に対する花弁の割合が大きい傾向があり、この方向性の違いはコサイン空間でより鮮明に現れる。そのためコサイン WTA はバージニカを完全に分離できる。

まとめると、コサイン WTA は特徴ベクトルの「形」（成分の比率）を基準にクラスタリングし、ユークリッド WTA はその「大きさ」（絶対値）を基準にする。Iris においてバーシカラーとバージニカの本質的な差は花弁の「比率」にあるため、コサイン基準の方が向いており、精度も高くなった（93.3% vs 88.0%）。

この比較からわかるのは、距離指標の選択はただの実装の細部ではなく、「データのどの側面を重視するか」という前提の違いだということである。特徴量の比率に意味があって絶対値はあまり関係ないデータには、L2 正規化とコサイン WTA の組み合わせを使う方が自然である。
