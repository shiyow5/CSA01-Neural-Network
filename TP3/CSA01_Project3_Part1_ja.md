## a) 解いた問題

本プロジェクトでは、**Hopfieldニューラルネットワーク**を自己連想記憶として実装した。
このネットワークに複数の2値パターンを記憶させ、ノイズで破損した入力から元のパターンを復元できるかどうかを検証することが目的である。

ネットワークの仕様は以下の通りである。
- **ニューロン数:** 120個（10行 × 12列のグリッド）

- **ニューロンの状態:** $s_i \in \{-1, +1\}$（$-1$が点灯画素、$+1$が背景）

ノイズの定義は、1から0、あるいは0から1への画素値の反転であり、実験では以下の3条件で検証した。
- **ノイズ率 0%:** 完全なパターンをそのまま入力
- **ノイズ率 10%:** 120画素中約12画素をランダム反転
- **ノイズ率 15%:** 120画素中約18画素をランダム反転

## b) アルゴリズムとネットワーク構造

### Hopfieldネットワークの概要

Hopfieldネットワークは、全ニューロンが相互に結合した再帰型ネットワークである。
記憶された各パターンがエネルギー関数の局所最小点に対応しており、ネットワークは入力パターンから出発してエネルギーを最小化しながら最も近い記憶へと収束する。

### 重み行列の学習（Hebbianルール）

重み行列は、記憶させたいパターン $\{\boldsymbol{\xi}^\mu\}$ から以下の式で一括計算する（ワンショット学習）。

$$W_{ij} = \frac{1}{N} \sum_{\mu=1}^{P} \xi_i^\mu \, \xi_j^\mu, \quad W_{ii} = 0$$

ここで $N = 120$ はニューロン数、$P$ は記憶パターン数である。
対角成分を0にするのは、ニューロンが自分自身の出力で自己強化されるのを防ぐためである。

### 非同期更新ルール

想起は**非同期更新**で行う。1つずつニューロンを順番に更新し、他のニューロンの現在の状態を使ってニューロン $i$ の新しい状態を決定する。

$$s_i \leftarrow \text{sign}\!\left(\sum_{j \neq i} W_{ij} \, s_j\right)$$

$\text{sign}(x) = +1$（$x \geq 0$ のとき）、$-1$（$x < 0$ のとき）とする。

### エネルギー関数と収束保証

ネットワークの動作はLyapunovエネルギー関数によって制御される。

$$E = -\frac{1}{2} \sum_{i \neq j} W_{ij} \, s_i \, s_j$$

非同期更新のたびにこのエネルギーは単調減少（または変化なし）することが数学的に保証されており、ネットワークは必ず有限ステップで固定点に収束する。

### 記憶容量

Hopfieldネットワークが信頼性よく記憶できるパターン数の上限はおよそ以下の通りである。

$$P_{\max} \approx 0.138 \times N = 0.138 \times 120 \approx 16 \text{ パターン}$$

今回は4パターンのみを記憶するため、容量面では余裕がある。

## f) 考察

### 「4」だけが正確に想起される理由

実験を通じて最も気になったのは、パターン2（「4」）だけが正確に想起される一方、パターン0（「0」）とパターン1（「2」）はノイズゼロの入力でさえ元のパターンに戻らないという点だった。

重み行列はHebbianルールにより記憶パターンの外積の和として計算されるため、パターン間の重なり（相関）がそのまま重みに反映される。相関が強いパターンどうしは重みベクトル空間で近い位置を占め、お互いの引力域を干渉し合う。

「0」と「2」はどちらも同じ列付近に上下の横断バーを持つため、画素の重なりが大きい。これが同じ擬似吸引子に収束する原因だと思われる。
「4」は左斜めの腕と右側の縦棒という形状が他の3パターンとかなり異なるため、対応する引力域が比較的独立して形成されているようだ。

### 擬似吸引子（Spurious Attractor）

「0」や「2」の想起が収束した状態は、記憶させた4パターンのいずれとも一致しない。これを**擬似吸引子**と呼ぶ。

Hopfieldネットワークを複数の相関パターンで学習すると、学習に使ったパターンの「混合状態」にあたるエネルギー的な極小点が重み行列の中に生まれることがある。今回の場合、「0」「2」「8」が互いに類似しているため、それらの平均的な形状に近い状態（横断バーを持つ「0」様構造）が擬似吸引子として形成された。一度この盆地に入ると、ネットワークはノイズ率に関わらずこの状態へ収束する。

### エネルギー関数と収束保証の意味

非同期更新のたびにエネルギーは単調減少することが保証されているため、ネットワークは必ず有限ステップで固定点へ収束する。実験では1〜4スイープで常に収束が確認された。
ただし、エネルギー最小化が保証するのは「何らかの固定点への収束」であり、「意図した記憶への収束」ではない。この点がHopfieldネットワークの本質的な限界である。

### ノイズに対する頑健性

ノイズ率が0%から15%に変化しても、収束先の固定点は変わらなかった。想起結果が間違った状態（擬似吸引子）であることには変わりないが、その「間違い方」はノイズ量によらず一定だった。引力域がかなり広いと考えられる。収束に要するスイープ数はノイズなしで1〜3回、15%ノイズで2〜4回とわずかに増加したが、収束自体は常に達成された。

### 容量と相関パターンの問題

120ニューロンのHopfieldネットワークの理論的容量はおよそ16パターンであり、今回の4パターンは容量の制限を大きく下回っている。それでも擬似吸引子が発生したのは、**容量ではなくパターン相関が問題だった**ことを示している。

ピクセルアートの数字パターンは、すべて同一の背景（+1）を共有しているため、Hebbianルールで計算された重みの中に強い「背景成分」が含まれる。この共通成分が特定のパターン間で干渉を引き起こす。これを改善する手法として平均減算前処理が考えられるが、Practiceセクションの実験eで検証した結果、この方法が±1 Hopfieldネットワークでは機能しないことが判明した。その理由と詳細はe)で述べる。

## d) 容量実験 — ネットワークは何パターンまで記憶できるか？

### 動機

Part 1では、「0」と「2」の想起が失敗した原因をパターン間の相関と特定した。
しかし、理論的容量（$P_{\max} \approx 16$）に対して4パターンしか記憶していないため、容量の問題ではないことは確認済みである。
ここでは**ランダム±1パターン**を使い、パターン数Pを増やしたときに想起精度がどのように変化するかを実測し、理論値と比較する。

### 実験設定

- ニューロン数: N = 120（Part 1と同じ）
- 記憶パターン数 P: 4, 8, 12, 16, 20, 24
- 各パターン: 各ニューロンで均等確率の±1ランダム値
- ノイズ率: 10%（12画素を反転）
- 試行数: Pの設定ごとに20種類の乱数セット
- 評価指標: 平均画素正解率 = (正しく想起された画素数) / 120

### ソースコード：`capacity_experiment.c`

```c
/************************************************************************************/
/* Hopfield network capacity experiment                                             */
/* Measures recall accuracy as number of stored patterns P increases               */
/* Uses random ±1 patterns and 10% noise for recall                                */
/* Extended by m5301059 SATO Sho.                                                   */
/************************************************************************************/
#include <stdlib.h>
#include <stdio.h>

#define N        120
#define MAX_P    24
#define NOISE    0.10
#define N_TRIALS 20
#define MAX_ITER 200

int    pat[MAX_P][N];
double w[N][N];
int    v[N];

void generate(int P, unsigned int seed)
{
    srand(seed);
    for (int mu = 0; mu < P; mu++)
        for (int i = 0; i < N; i++)
            pat[mu][i] = (rand() % 2 == 0) ? 1 : -1;
}

void store(int P)
{
    for (int i = 0; i < N; i++) {
        for (int j = 0; j < N; j++) {
            w[i][j] = 0.0;
            for (int mu = 0; mu < P; mu++)
                w[i][j] += (double)pat[mu][i] * pat[mu][j];
            w[i][j] /= (double)P;
        }
        w[i][i] = 0.0;
    }
}

double recall(int m, unsigned int noise_seed)
{
    srand(noise_seed);
    for (int i = 0; i < N; i++) {
        double r = (double)(rand() % 10001) / 10000.0;
        v[i] = (r < NOISE) ? -pat[m][i] : pat[m][i];
    }
    for (int iter = 0; iter < MAX_ITER; iter++) {
        int changed = 0;
        for (int i = 0; i < N; i++) {
            double net = 0.0;
            for (int j = 0; j < N; j++)
                net += w[i][j] * v[j];
            int vn = (net >= 0) ? 1 : -1;
            if (vn != v[i]) { v[i] = vn; changed++; }
        }
        if (!changed) break;
    }
    int correct = 0;
    for (int i = 0; i < N; i++)
        if (v[i] == pat[m][i]) correct++;
    return (double)correct / N;
}

int main()
{
    int ps[] = {4, 8, 12, 16, 20, 24};
    int np   = 6;
    printf("P,avg_accuracy\n");
    for (int pi = 0; pi < np; pi++) {
        int P = ps[pi]; double total = 0.0; int count = 0;
        for (int t = 0; t < N_TRIALS; t++) {
            generate(P, t * 137 + 42);
            store(P);
            for (int mu = 0; mu < P; mu++) {
                total += recall(mu, t * 137 + mu + 1000);
                count++;
            }
        }
        printf("%d,%.4f\n", P, total / count);
    }
    return 0;
}
```

### 考察

理論的な臨界点P = 16を超えても、精度は急激に崩壊せず**緩やかに低下**した。
P = 16での正解率は97.5%と依然高く、P = 24（理論限界の45%超）でも86%を維持している。

この緩やかな劣化は、理論容量$P_{\max} = 0.138N$が**最悪ケースの上限値**として導出されているためである。
ランダム±1パターンはNが大きいとき平均的にほぼ直交しており（内積の期待値が$O(1/\sqrt{N})$に収まる）、クロストーク雑音が理論の最悪想定より小さく抑えられる。

## e) 平均減算前処理 — 擬似吸引子を修正する試み

### 動機

Part 1で「0」と「2」が擬似吸引子へ収束した根本原因は、4パターンすべてが+1の背景を共有しているためである。
この共通成分は重み行列に以下のバイアス項を生じさせる。

$$W_{ij}^{\text{bg}} = \frac{1}{N} \sum_\mu \xi_i^{\mu,\text{bg}} \xi_j^{\mu,\text{bg}} = \frac{P}{N}$$

この正の定数バイアスは、あたかも「すべて+1」のダミーパターンが余分に記憶されているかのように作用し、想起を背景方向へ引き寄せる。
これを除去するための前処理として、学習前に各パターンから平均パターンを引く方法が考えられる。

$$\tilde{\xi}_i^\mu = \xi_i^\mu - \bar{\xi}_i, \quad \bar{\xi}_i = \frac{1}{P}\sum_{\mu=1}^P \xi_i^\mu$$

$$W_{ij} = \frac{1}{P}\sum_\mu \tilde{\xi}_i^\mu \tilde{\xi}_j^\mu, \quad W_{ii} = 0$$

すべてのパターンで同じ値を持つ画素は$\tilde{\xi}_i^\mu = 0$となり、重み行列への寄与がゼロになる。

### ソースコード：`mean_subtraction.c`

```c
/************************************************************************************/
/* Mean subtraction preprocessing for Hopfield network                             */
/* Extended by m5301059 SATO Sho.                                                   */
/************************************************************************************/
#include <stdlib.h>
#include <stdio.h>

#define N     120
#define P     4
/* (パターン定義はPart 1と同じため省略) */

double w_orig[N][N], w_sub[N][N];
double mean_p[N], xi_sub[P][N];
int v[N];

void compute_mean() {
    for (int i = 0; i < N; i++) {
        mean_p[i] = 0.0;
        for (int mu = 0; mu < P; mu++) mean_p[i] += pat[mu][i];
        mean_p[i] /= (double)P;
    }
}

void store_sub() {
    for (int mu = 0; mu < P; mu++)
        for (int i = 0; i < N; i++)
            xi_sub[mu][i] = pat[mu][i] - mean_p[i];
    for (int i = 0; i < N; i++) {
        for (int j = 0; j < N; j++) {
            w_sub[i][j] = 0.0;
            for (int mu = 0; mu < P; mu++)
                w_sub[i][j] += xi_sub[mu][i] * xi_sub[mu][j];
            w_sub[i][j] /= (double)P;
        }
        w_sub[i][i] = 0.0;
    }
}
```

### 実験結果（ノイズ10%）

| パターン | 前処理なし（HD） | 平均減算あり（HD） |
|:-------:|:------------:|:--------------:|
| 「0」 | 9  | **22**（悪化） |
| 「2」 | 17 | 17（変化なし） |
| 「4」 | **0**（完全一致） | **11**（崩壊） |
| 「8」 | 0  | **11**（崩壊） |

*HD = 記憶パターンとのHamming距離。小さいほど良い。*

### 予想外の結果

平均減算は想起精度を**改善せず、むしろ悪化させた**。
前処理なしで完全に想起できていた「4」と「8」すら崩壊し（HD = 11）、「0」の想起結果は原型をとどめない。

### なぜ平均減算が機能しないのか

失敗の根本原因は、**前処理後の重み空間と±1状態空間の不整合**にある。

平均減算後の有効パターン$\tilde{\boldsymbol{\xi}}^\mu$は±1ではなく連続値（約$-2$から$+2$の範囲）をとる。
このような連続値パターンから構築された重み行列は、元の±1パターンを固定点として持たない。
つまり、更新ルール$s_i \leftarrow \text{sign}(\sum_j W_{ij} s_j)$において、$\boldsymbol{\xi}^\mu$は必ずしも不動点にならない。

$$\text{sign}\!\left(\sum_j W_{ij}^{\text{sub}} \cdot \xi_j^\mu\right) \neq \xi_i^\mu$$

平均減算後のエネルギー地形の極小点は、記憶させた数字パターンのどれとも対応しない。
もし減算後にさらに$\text{sign}()$で±1に二値化すれば整合するが、背景画素が多い数字パターンでは多くの画素が0付近となり情報が失われる。

正しい解決策はHebbianルールではなく、**擬似逆行列学習則**（Pseudo-inverse rule）の使用であり、これにより任意の相関を持つパターンを不動点として明示的に保証できる。

## f) 考察：容量・相関・Hebbian学習の限界

### 2種類の想起失敗モード

Part 1、d、eの実験から、Hopfieldネットワークの想起失敗には独立した2種類の原因があることがわかった。

| 失敗モード | 原因 | 観察条件 |
|---------|------|---------|
| 容量超過 | Pが多すぎてクロストーク雑音が支配的になる | P > 0.138N（実験d） |
| 相関雑音 | パターンが共通成分を持ち擬似吸引子が生じる | P = 4の数字パターン（Part 1・実験e） |

この2つは**独立**した問題である。
実験dでは24個のランダムパターン（理論限界超過）でも86%の精度を保てた。
一方Part 1ではわずか4パターンで想起失敗が起きた。失敗の原因はパターン数ではなくパターンの構造的相関にある。

### ランダムパターンと構造的パターンの違い

ランダム±1パターンはNが大きいとき互いにほぼ直交する。

$$\langle \boldsymbol{\xi}^\mu \cdot \boldsymbol{\xi}^\nu \rangle = 0 \quad (\mu \neq \nu)$$

このため重み行列は各パターンの外積の線形和として近似でき、クロストーク干渉が小さい。
理論容量はこの直交性を前提として導かれており、実測でも緩やかな劣化（P=24で86%）として現れた。

数字ピクセルアートパターンは直交性がはるかに低い（内積がNの70〜80%に達することもある）。
このため、理論容量の観点では余裕があるはずのP = 4でも、実効的な記憶容量を超えた状態になっている。

### Hebbian学習が相関に対して本質的に無力な理由

Hebbianルールはワンショット学習であり、新パターンを追加するたびに外積を足すだけで、既存パターンの不動点条件を確認しない。
パターン間の相関が高い場合、後から追加したパターンの外積が以前のパターンの引力域を破壊し、擬似吸引子を生み出す。
実験eは、最も自然な前処理（平均減算）でもこの問題が解決できないことを示した。

**根本的な解決策：**

- **擬似逆行列学習則:** $W = \Xi^+ (\Xi^+)^T$（$\Xi^+$はパターン行列の擬似逆行列）。パターン間の相関に関わらず不動点を保証するが計算コストが高い。
- **現代Hopfieldネットワーク（Ramsauer et al., 2020）:** エネルギー関数を指数関数化することで容量を指数的に拡大（最大$2^{N/2}$パターン）し、相関パターンへの対応能力も向上する。

### まとめ

| 実験 | 主な知見 |
|-----|---------|
| Part 1 | ランダムシード頼りの固定ネットワークで、構造的パターンは擬似吸引子に収束 |
| d) 容量 | 理論限界付近でも緩やかに劣化。ランダムパターンではHebbian則が有効 |
| e) 平均減算 | 直感的な前処理でも±1 Hopfieldとは空間的に不整合。相関問題の根深さを示す |

パターン数が少なく互いにほぼ直交している場合、Hopfieldネットワークは十分に機能することが確認できた。
一方、共通の背景を持つ数字・文字のような構造化パターンに対しては、Hebbianルールの限界が早い段階で現れ、理論容量を大きく下回る実効容量しか得られない。
この問題を根本的に解決するには、擬似逆行列学習則や現代Hopfieldネットワークのような別の手法が必要になると考えられる。
